Slide 5: Process — Stacking Ensemble ทำงานยังไง

ทฤษฎี Ensemble Learning: "โมเดลหลายตัวรวมกัน แม่นกว่าตัวเดียว" (Wisdom of Models)

ขั้นตอน Stacking: ``` ขั้น 1: แบ่ง Training Data เป็น 5 ส่วน (5-fold CV)

ขั้น 2: แต่ละโมเดลฝึก 4 ส่วน ทำนาย 1 ส่วน (สลับรอบ) ┌─ GradientBoosting → ทำนาย Prob [H, D, A] ├─ XGBoost → ทำนาย Prob [H, D, A] └─ RandomForest → ทำนาย Prob [H, D, A]

ขั้น 3: Logistic Regression (Meta-Learner) เรียนรู้ว่า "เมื่อไหร่ควรเชื่อโมเดลไหน"

ขั้น 4: ผลลัพธ์สุดท้าย = คำตอบที่ดีที่สุดจาก 3 โมเดลรวมกัน ```

ทำไมถึงดีกว่าโมเดลเดียว? - GB เก่งเรื่องจับ pattern ต่อเนื่อง (Sequential) - RF เก่งเรื่องความเสถียร (Bagging) - XGBoost เก่งเรื่อง Regularization (ลด Overfitting) - LR เรียนรู้ว่าเมื่อไหร่โมเดลไหน "ถูก" บ่อยกว่า

Slide 7: Experiment & Analysis

เราทดลอง 5 แนวทาง แล้ววัดผลบน 380 นัดจริง:

# วิธี Accuracy สถานะ
1 Random Forest (Baseline AI) 52.37% ❌
2 GradientBoosting + FPL + Odds 53.95% ❌
3 Dixon-Coles (Poisson Regression) 45.53% ❌
4 NLP Text Features (Guardian) +0.26% ❌
5 Stacking Ensemble (GB+XGB+RF→LR) 55.26% ✅

วิเคราะห์: - Dixon-Coles ไม่ช่วย → ซ้ำซ้อนกับ Elo (จับ "ความแข็งแกร่ง" เหมือนกัน) - NLP ช่วยน้อย → ขาดบทความสำหรับ training data ย้อนหลัง - Stacking คือคำตอบ → ใช้จุดแข็งของหลายโมเดลร่วมกัน
